Plan for PmergeMe:


STAGE 1: Input Parsing & Validation
    Goal: Convert incoming command-line strings into raw tracking containers.

    Execution: Loop through argv beginning at index 1. Split compound strings if numbers are space-separated, check that every character is a digit,
               ensure values are positive integers, and verify no integer overflows occur.


STAGE 2: Implementation for Container A (std::vector)
    Goal: Sort elements using a dedicated vector pairing logic.

    Execution:
        Step 2.1: Block Level Pairing: Instead of treating elements individually, the recursion will accept a blockSize parameter.
                  At depth 0, blockSize = 1. At depth 1, blockSize = 2. I will loop through the flat _vectorData array and swap adjacent chunks of memory to ensure the larger chunk (determined by its final element) is on the right.

        Step 2.2: Recursive Sorting: Call _sortVector(blockSize * 2). When this function returns, the "winning" chunks inside the flat array are guaranteed to be sorted.

        Step 2.3: Parallel Extraction: Create two empty vectors of integers: mainChain and pendingChain.
                  Crucially, create two corresponding size_t vectors: mainIDs and pendingIDs.
                  Iterate through _vectorData by blockSize, extracting the "winner" chunks into mainChain and the "loser" chunks into pendingChain,
                  assigning them matching IDs so they never lose their paired relationship.

        Step 2.4: Custom Binary Search: Standard std::lower_bound cannot step by blockSize chunks.
                  I will write a custom binary search that calculates midpoints based on block boundaries and only compares the final elements of those blocks.

        Step 2.5: Jacobsthal Insertion: Generate the Jacobsthal index sequence. Iterate through the pendingChain in reverse blocks.
                  For each element, look up its target ID in mainIDs to establish the strict upper bound, execute the custom binary search, and insert the block and its ID into mainChain and mainIDs.

        Step 2.6: Flattening: Copy the fully assembled mainChain back into _vectorData, overwriting the memory with the newly sorted state for this recursion depth.


STAGE 3: Duplicating Implementation for Container B (std::deque)
    Goal: Avoid generic template shortcuts to satisfy the instruction: "It is strongly advised to implement algorithm for each container and thus to avoid using a generic function."
    Execution: Replicate the exact geometric logic from Stage 2 using a std::deque<std::pair<int, int> > and a std::deque<int> for the main chain.


STAGE 4: Instrumentation and Precise Timing Output
    Goal: Capture data preparation and sorting metrics.
    Execution: Wrap  execution steps in high-precision microsecond timing blocks. In C++98, this is best accomplished using gettimeofday from <sys/time.h>.
